# Editorial Writing

## Voice

Write in Spanish for a user who scans, worries about fake downloads, and wants the official route quickly.

Tone:

- clear;
- practical;
- editorial, not marketing;
- honest about uncertainty;
- specific to the tool.

## Structure

Every ficha should answer:

1. What is this tool?
2. Can I download it, or is it web/extension/store only?
3. Which official channel should I use for my platform?
4. What scams or false installers should I avoid?
5. When is this tool a good choice?
6. What alternatives should I compare?

## SEO Without Thin Content

Use keywords naturally:

- `descargar <tool>`;
- `<tool> para Windows/Mac/Android/iOS`;
- `app oficial`;
- `instalador oficial`;
- `web oficial`;
- `APK`;
- `alternativas a <tool>`;
- category phrases like `generador de imágenes IA`, `modelo local`, `asistente IA`, `video IA`, `copiloto de programación`.

Do not keyword-stuff. Put the user’s real confusion into the text.

## Editorial Sections

Use 3-5 sections. Each section should have:

- a specific heading;
- two short paragraphs;
- at least one concrete decision or warning.

Good section ideas:

- “Por qué tanta gente busca descargar X”
- “X no se instala como una app tradicional”
- “Web, escritorio y móvil: qué canal usar”
- “Cómo evitar clones o instaladores falsos”
- “Cuándo elegir X y cuándo mirar alternativas”

## FAQ

FAQ should target search intent and schema:

- “¿X tiene app oficial para Windows?”
- “¿Es seguro descargar X desde terceros?”
- “¿X funciona en español?”
- “¿Necesito cuenta para usar X?”
- “¿Existe APK oficial de X?”

Answers must be concise and must not overpromise.
